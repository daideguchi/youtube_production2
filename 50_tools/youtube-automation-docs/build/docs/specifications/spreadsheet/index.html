データ検証ルール​

必須列チェック​

# qwen.md に記載される検証ロジック
required_columns = [
    "No.",
    "タイトル",
    "サムネタイトル",
    "企画意図",
    "ターゲット層",
    "具体的な内容",
    "DALL-Eプロンプト"
]

def validate_row(row):
    """
    各行の必須列が埋まっているかチェック
    """
    for col in required_columns:
        if not row[col] or row[col].strip() == "":
            return False, f"列 '{col}' が空です"

    # No.の重複チェック
    if is_duplicate_no(row["No."]):
        return False, f"No. {row['No.']} が重複しています"

    return True, "OK"

台本制約チェック​

def validate_script(script):
    """
    台本が整形可能かチェック
    """
    # 読点過多チェック
    sentences = script.split("。")
    avg_commas = sum(s.count("、") for s in sentences) / len(sentences)
    if avg_commas > 1.2:
        return False, f"読点過多: 平均 {avg_commas:.1f}個/文（上限1.2）"

    # 整形可能性チェック（概算）
    if not can_format_to_27x2(script):
        return False, "27×2行整形不可（文長・改行位置）"

    return True, "OK"

運用上の注意点​

🚨 AI不可領域​

以下の列は絶対にAIが書き換えてはいけない:

  • 人間メモ
  • 投稿指示 （人間が設定）
  • 台本OK承認 （人間が明示的に変更）

💡 ベストプラクティス​

  1. 朝一で30本入力 - 必須列を一気に埋める
  2. バッチ起動前に目視確認 - 重複No.、欠損列チェック
  3. 台本承認は集中レビュー - 30本×2分=1時間を確保
  4. エラーは即修正 - 黄色・赤色ステータスを優先対応
  5. DriveURLは必ず確認 - 動画が正しく保存されているか
