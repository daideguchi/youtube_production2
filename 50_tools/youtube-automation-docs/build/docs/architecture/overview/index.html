YouTube自動化システム 設計概要

システム概要​

スプレッドシート中心の生きたアーキテクチャ - 30本/日の動画生成を完全自動化

コアコンセプト​

1. スプレッドシート = 唯一の真実​

  • 30本/日の企画を1シートで管理
  • 進捗列でステータス管理（企画入力→台本OK→音声OK→画像OK→投稿済）
  • 人が触る欄とAI実行欄を明確分離

2. 細密な制約 = 品質保証​

台本整形:
  - 1セクション: 最大2行
  - 1行: 27文字以内
  - セクション末尾: "///" 必須
  - 読点: 平均1.2個/文以下

SRT処理:
  - タイムスタンプ: 絶対不変
  - インデックス: 絶対不変
  - チャンク: 10-20秒の意味単位

映像制約:
  - セーフマージン: 10%厳守
  - 座標: テンプレート丸め値

3. チャンネル別プリセット（解説系特化）​

品質保証（DoD）​

各段階の Definition of Done（合格基準）:

台本生成​

  • ✅ 自然な標準語（芝居がかった表現なし）
  • ✅ 読点過多なし（平均1.2個/文以下）
  • ✅ 短文改行で可読性確保

SRT整形​

  • ✅ 27文字×2行でセクション化
  • ✅ セクション末尾に ///
  • ✅ タイムスタンプ・インデックス不変

画像生成​

  • ✅ チャンネルスタイル準拠
  • ✅ セーフマージン10%
  • ✅ 文字と被写体の衝突ゼロ

CapCut自動制作​

  • ✅ 音声・字幕・画像の完全統合
  • ✅ 座標・秒数テンプレート丸め
  • ✅ タイムライン自動生成

投稿​

  • ✅ DriveURL記録済
  • ✅ メタデータ（タイトル・説明・タグ）完備
  • ✅ 人による最終ゲート承認済

日次運用フロー​

所要時間見積もり:

  • 企画入力: 30分（30本×1分）
  • バッチ実行: 2-3時間（自動・並列処理）
  • 人間確認: 1時間（30本×2分）
  • 合計: 約4時間で30本完成
