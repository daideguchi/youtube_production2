# OPS_A_TEXT_TECHNIQUE_PACKAGES.md

この文書は、Aテキスト台本に「効くテクニック」を“パッケージ（モジュール）”として再利用できる形で固定するためのSSOTです。

## 1) 使い方（運用）
- 目的: あるチャンネル/台本に「同じ勝ち筋の技法」を安全に差し込める状態にする。
- 適用方法（標準）:
  1. 適用したいモジュールを選ぶ（下記）。
  2. チャンネルの `script_prompt.txt` に **モジュールの指示ブロックをコピペ**（チャンネル固有ルールと矛盾しない位置に配置）。
  3. Portal表示ズレ防止のため、同チャンネルの `channel_info.json.script_prompt` も **必ず同内容に同期**する。

## 2) モジュール一覧
- `M01_invisible_hook_payoff_catharsis_v1`: 冒頭に仕掛けを置き、ゆっくり回収して最後にカタルシス（視聴者はテクニックに気づかない自然さ）。

---

## M01_invisible_hook_payoff_catharsis_v1

### 狙い
視聴者が「意識して追っていないのに、最後に全部つながって泣ける/救われる」状態を作る。

### 適用条件（向いている）
- 物語・一人語り・内省が主役の台本（癒し/仏教/人生/シニア系など）。
- “驚きの種”より“静かな腑に落ち”を価値にしたい回。

### 向いていないケース
- 章見出し禁止など、構造が極端に制約されている場合（適用はできるが、回収ポイントの設計が難しくなる）。
- 断定/結論の強い解説系（カタルシスより納得を優先する回）。

### コピペ用：指示ブロック（プロンプトに入れる）
```text
【ナラティブ技法（必須）: さりげない仕掛け→ゆっくり回収→カタルシス】
- 台本の最初（オープニングの最初の1〜2段落）に、説明しない“鍵”を1つだけ置く。
  - 鍵は、日常の小物/音/匂い/癖/短い言葉など、自然に紛れるものにする（象徴として解説しない）。
- 中盤〜終盤にかけて、その鍵に「同じ言葉/同じ触感/同じ景色」として2〜3回だけ自然に触れる。
  - 露骨な回想（「覚えていますか」等）や、わざとらしい強調はしない。
  - “気づいた人だけ得”の薄さで、ただ同じ空気が戻ってくる感じにする。
- 最後（クロージング直前〜最後の2段落）で、鍵が静かに意味を変える形で回収し、小さなカタルシスを作る。
  - カタルシスは大げさな種明かしではなく、「ああ、そうだったのか」という安堵/尊厳/救いに落とす。
- 禁止: 本文内に「伏線」「回収」「仕掛け」などのメタ語を書かない（設計思想は本文に出さない）。
```

### QA（セルフチェック）
- [ ] 鍵は冒頭の1〜2段落に置けている（説明しすぎていない）。
- [ ] 鍵への触れ直しは2〜3回で、わざとらしい回想になっていない。
- [ ] 最後に“意味が変わる”回収があり、安堵/救いのカタルシスで終われている。
