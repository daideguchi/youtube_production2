# HISTORY: UIフロント（ui-frontend）`node_modules` 復旧メモ（2026-01-26）

## 背景 / Background
- **Mac-first（Hot=未投稿/編集中はMacに実体）**の前提で、進捗確認やボード確認のため **UIが止まるのはNG**。
- 省容量対応などで `apps/ui-frontend/node_modules` が削除/空になった場合、`react-scripts` の依存が解決できず UI が起動できない。

## 症状 / Symptom
- `apps/ui-frontend` の開発サーバで以下のようなエラーが出る:
  - `Module not found: ... @pmmmwh/react-refresh-webpack-plugin ...`
  - `Module not found: ... babel-loader ...`
  - `Module not found: ... webpack-dev-server ...`
  - `Html Webpack Plugin: ... html-webpack-plugin ...`

## 原因 / Root cause
- `apps/ui-frontend/node_modules` が実質空（例: `.cache` しか無い）で、Webpack/CRA 周辺の依存が解決できない。

## 対応 / Fix（lockfile準拠）
```bash
cd apps/ui-frontend
npm ci --no-audit --no-fund
```

### 検証 / Verify
```bash
cd apps/ui-frontend
npm run build
```

## 注意 / Notes
- `node_modules` は **Git管理しない**（復旧は `npm ci`）。
- `npm ci` は `package-lock.json` を正とし、`node_modules` を作り直す。
- 開発サーバを起動中だった場合は、復旧後に `npm start` を再起動する。
