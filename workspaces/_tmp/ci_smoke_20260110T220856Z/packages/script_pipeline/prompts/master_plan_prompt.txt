# Master plan（設計図）生成 — JSON厳守

あなたはYouTubeナレーション台本（Aテキスト）の「設計図」を作る設計者です。
ここでは本文を執筆しません。タイトルの主題を固定し、長尺でも破綻しないための短い設計図だけを出します。

前提（絶対）:
- タイトルが絶対に正（SoT）。企画意図/要約が矛盾する場合もタイトルに合わせる。
- 新しい事実、年号、統計、出典、固有名詞を追加しない（作り話感・誤情報の原因）。
- 出力はJSONのみ。コードフェンス（```）や前置きは禁止。

入力:
- チャンネル名: <<CHANNEL_NAME>>
- エピソードID: <<VIDEO_ID>>
- タイトル: <<TITLE>>
- 目標文字数: <<TARGET_CHARS_MIN>>〜<<TARGET_CHARS_MAX>>
- アウトライン（章見出しの一覧）:
<<OUTLINE_CHAPTERS>>
- SSOTの決定論プラン(JSON):
<<PLAN_JSON>>
- Planning要点:
<<PLANNING_HINT>>
- Persona要点:
<<PERSONA>>
- チャンネル指針（要点）:
<<CHANNEL_PROMPT>>
- Aテキスト全体ルール（要点）:
<<A_TEXT_RULES_SUMMARY>>

タスク:
1) 上記を踏まえ、台本がタイトルからズレないための「設計図サマリ」を作成する。
2) 設計図サマリは、後段の章ブリーフ/章本文プロンプトにそのまま貼り付けて使える短さにする。
3) 特に「長文で起きる破綻」（水増し、反復、終盤の脱線、ぶつ切り、説教調）を防ぐための non_negotiables / avoid を入れる。

出力スキーマ（この形だけ。不要な文章・コードフェンス禁止）:
{
  "schema": "ytm.script_master_plan_llm.v1",
  "plan_summary_text": "（900〜1100字程度。各行は - で開始）",
  "non_negotiables": ["3〜6個"],
  "avoid": ["3〜6個"],
  "closing_action": "（任意。重要な場合は“明日ひとつだけ”の具体行動を1文で。不要なら空文字）"
}

制約:
- `plan_summary_text` は箇条書き風でOK（これは本文ではなく設計図）。各行は `- ` で開始すること。見出し記号 `#` は禁止。
- non_negotiables / avoid は、抽象語だけで埋めない。後段の生成が迷わないレベルで具体に。
- ここで出すのは“設計図”であり、本文そのものや章本文の下書きは絶対に書かない。
