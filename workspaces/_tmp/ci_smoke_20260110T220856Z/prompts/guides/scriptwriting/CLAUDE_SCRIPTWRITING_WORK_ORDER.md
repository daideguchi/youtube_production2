# CLAUDE_SCRIPTWRITING_WORK_ORDER — 「調査はOrchestrator / 執筆はClaude」分業の作業指示書

目的:
- 実態調査（企画CSV・既存回との重複・禁則違反・字数不足など）を **Orchestrator が確定**し、Claude を **執筆だけ**に専念させる。
- 「企画で決めた趣旨」と「台本本文」の不一致、同テーマ乱立、Aテキスト禁則違反、等間隔 `---` 乱打を防ぐ。

位置づけ:
- ここは `prompts/` 配下の **運用ガイド**。プロンプト本文の正本（SoT）は `packages/**/prompts/` 側。
- チャンネル別の参照先は `prompts/guides/scriptwriting/channels/CHxx.md` を正として参照する（複製しない）。

NOTE（2026-01-09）:
- 台本3パターン（Planning の `台本型` / kata1〜3）運用は廃止。Work Order に「型」を書かない。

---

## 0) 役割分担（強制）

### Orchestrator（あなた / このリポジトリ側で実施）
- 企画の正本（Planning SoT）を読み、**趣旨を確定**する。
- 既存回との重複を調べ、**差別化軸を1つに固定**する。
- 禁則/字数/区切り/捏造リスクを点検し、**直すべき箇所を明文化**する。
- Claude に渡す「作業指示（=Work Order）」を **完成形で作る**（Claudeに調査させない）。

### Claude（外部LLM）
- Work Order に基づき **台本本文（Aテキスト）の執筆/修正のみ**を行う。
- 調査・検索・推測補完・勝手な企画改変はしない。
- 指示が「言い回しの自然化のみ」の場合は、内容（論点/構成/主張）を動かさず、文章だけを自然に整える（硬さ/同語反復/語尾連打/回りくどさを削る）。
  - この場合、下書き本文を最大限残し、基本は追記で厚みを作る（削除/差し替えは必要最小限）。

---

## 1) Orchestrator が集めて確定するもの（チェックリスト）

最低限（必須）:
- チャンネル: `CHxx`
- 動画番号 / 動画ID
- タイトル（最優先で正）
- 企画意図（何を救い、視聴後に何が変わるか）
- ターゲット層（Personaの固定文 or 指定文）
- 構成案メモ（あるならそのまま）
- 目標文字数（`configs/sources.yaml`）
- チャンネル固有の必須/NG/トーン（SoTから **2〜6行に圧縮**して抜粋）

整合性（必須）:
- 既存回（特に投稿済み/公開済み）との類似候補（番号+タイトルを3〜8件）
- 差別化軸（今回だけの新規価値を1行で固定）
  - 例: 「同じ“孤独”でも、今日は“夜中に襲う不安”にだけ焦点を当て、対処は呼吸と言葉の置き換えに絞る」

修正対象がある場合（必須）:
- 修正対象の台本本文（全文）
- 問題点の事実（字数、`---` 回数、逸脱箇所、重複箇所）を短く列挙

推奨（あると強い）:
- 採用する仏教の教え（1つ）と、日常語の言い換え（2行）
- 使う史実・逸話（1つまで）の要約（年号断定/出典断定は避けた安全表現）
- 科学/心理の橋渡し（入れるなら安全な一般論を短く。数字・機関名なし）

---

## 2) Orchestrator の調査手順（最短ルート）

1) チャンネルガイドを開いて SoT パスを確定
- `prompts/guides/scriptwriting/channels/CHxx.md`

2) 企画CSVの該当行を抜き出す（タイトル/意図/構成案/型）
- `workspaces/planning/channels/CHxx.csv`

3) 重複・汚染を機械で一次検知（推奨）
- `python scripts/ops/planning_lint.py --channel CHxx`

4) 修正対象台本の実態確認（字数/禁則/`---`）
- `workspaces/scripts/CHxx/NNN/content/assembled.md`

5) 差別化軸を1行に固定して、Claudeに渡す（ここが最重要）
- 既存回と被るテーマはゼロにはできない前提で、「今回だけの角度」を固定する。

---

## 3) Claude に渡す Work Order（コピペ用テンプレ）

下の `<<WORK ORDER>>` だけを Claude に渡す。
（このテンプレ外の説明、CSVの行貼り付け、URL、長いメモは入れない。汚染源になる。）

```text
<<WORK ORDER>>
あなたはYouTubeチャンネルの台本執筆者です。あなたの仕事は「執筆のみ」です。調査・検索・推測補完はしません。

【最優先】
タイトルが絶対に正。タイトルと矛盾する情報は捨てる。

【チャンネル/案件】
チャンネル: CHxx
動画番号/動画ID: CHxx-NNN
タイトル: （ここにタイトル）

【企画の狙い（確定）】
企画意図: （ここに1〜2文）
ターゲット層: （ここに固定文）
差別化軸（今回だけの新規価値）: （ここに1行）
チャンネル固有の必須/NG/トーン（抜粋）:
（ここに2〜6行。例: “物語→教え→今夜の実践2〜3” 等）

【既存回との重複回避（確定）】
類似候補（番号+タイトル）:
1) （例: CHxx-123 ...）
2) ...
注意: 上の類似候補と「同じ結論・同じ例・同じ教えの見せ方」にならないよう、角度と具体例と実践を変える。

【字数（確定）】
目標文字数: NNNN〜NNNN

【採用する教え（確定）】
キーコンセプト: （例: 知足）
日常語の言い換え: （2行まで）

【使ってよい素材（確定）】
使ってよい逸話/史実の要約（任意。ある場合のみ）:
（3〜8行程度で安全表現。年号・出典断定なし）

【修正対象（ある場合のみ）】
下書き台本:
（ここに全文）

【今回の作業】
1. タイトル趣旨と本文を一致させる。逸脱段落は削除/差し替え。
2. 台本内の重複（言い換え水増し）を削り、厚みは具体で作る。
3. 目標字数まで増やす。ただし一般論の引き伸ばしは禁止。
4. `---` は文脈ベースで最小限。等間隔で入れない。

【Aテキスト絶対条件（破ったら失敗）】
- 出力は台本本文のみ。要約/アウトライン/制作メモ/チェックリスト/箇条書きは一切出さない。
- URLなし。
- 丸括弧（ ）と半角()は0個。
- 箇条書き/番号リスト/見出し行/Markdown装飾は0。
- 捏造統計・割合・研究機関名・論文名・年号断定を書かない。
- 最後まで完結させる。

【不足がある場合】
このWork Orderに「必須情報の欠落」がある場合のみ、質問を最大3つまでして止まる（本文はまだ出さない）。
欠落がない場合は質問せず、完成台本を出す。
<<WORK ORDER END>>
```

## 4) 運用メモ（事故を減らすコツ）

- Claude に「チャンネルの master prompt 全文」や「CSV行の長文コピペ」を渡さない。混ざると高確率で内容汚染や禁則違反が起きる。必要なチャンネル固有要件は **2〜6行に圧縮して抜粋**して渡す。
- 差別化軸は「言い回し」ではなく「何を扱い、何を捨てるか」を固定する。
- 史実・逸話を使うなら、Orchestrator が要約して渡す（Claudeに思い出させない）。
- 修正作業では「残す骨」と「捨てる肉」を最初に指示する（例: “導入の問いは残すが、人物例は全差し替え”）。
