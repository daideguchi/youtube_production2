## 3. Claudeの学習ポイント（187_修正履歴.md, 187_深い学習記録.md, AI音声YouTube台本作成ガイド.md）
### 3.1 音声台本のコア原則
1. 読みやすさ≠聴きやすさ: 主語はシーン説明で明示、会話では極力省略。
2. 量より質の深掘り: 複数箇所を浅く伸ばすのではなく、2-3シーンを徹底描写して時間を稼ぐ。
3. ポジティブフレーミング: 「避ける」「守る」より「得られる」「格が上がる」。4. 完全日常語化: 60代が初めて聞いても理解できる語彙に必ず変換。
5. 物語優先: 構造で切り刻まず、ドラマとして一気に聞かせる。
6. 語尾多様化: 同一語尾3連続NG。語尾の種類とリズムをチェックリスト化。
7. 長さ: 6,000-8,000字／15-20分を想定し、実際の音声時間で検証。

### 3.2 修正履歴からの具体知見
- 修正18: 時間拡張は「清水の30年越しの対比」や「優波離の生活描写」のように具体的シーンで行う。
- 修正19: 名前の音声衝突を避ける（例: 山田→石川）。音の頭子音・音節数をずらす。
- 修正17: サンスクリット固有名はカタカナ統一（優波離→ウパーリ）。
- 修正16: 主語が曖昧な連続会話は徹底的に洗い出して是正。
- 修正23: 会話シーンの主語削除でテンポを作り、ナレーションで補う。
- 修正24: 語尾の単調さは聴覚疲労を招くため、3種類以上の語尾をローテーション。

### 3.3 プロセス面の教訓
- 7ステップフロー（企画→初稿→音声最適化→感情強度チェック→仏教統合→実践セクション→クロージング）を守る。
- 各ステップでチェックボックスを設け、スコア100/100を狙う品質運用を継承。

## 4. ベンチマーク「逆転の一言」の学び（戦略.md / ベンチマーク参考）
1. 主人公人格化: 名前・年齢・職歴・価値観を冒頭で提示し、視聴者が「自分の物語」と錯覚できるようにする。
2. 敵役の明確化: 若い田島のように動機や口癖を描き、攻撃セリフを視聴者が実際に浴びる強度へ。
3. 感情の段階描写: 「心の中だけで涙」「胸には怒りと羞恥」など visceral な比喩を惜しまない。
4. 逆転の瞬間: 「学ばせてもらえますか？」のように、謙虚さで空気を一変させる言葉＋周囲の静寂・表情を描写。
5. 仏教統合: 危機の刹那に教えを想起させ、説教ではなく救済として働かせる。
6. 余韻: 後日談や謝罪シーンまで描き、癒しと希望で締める。
7. 文字数: ベンチマークは約6,000-8,000字。構成のフレーム（共感→事件→葛藤→逆転→癒し→希望）を踏襲する。

## 5. 制作フロー & チェックリスト
### 5.1 7ステップ実務
1. 企画・構成: CSVからテーマ/ターゲット/メッセージを決定、ペルソナメモを作る。
2. 初稿: 物語→教え→実践→エンディングの4パートで6,000字目安。
3. AI音声最適化: 主語ルール、語尾多様化、句読点配置、声優用のポーズ指示を整える。
4. 感情強度レビュー: 攻撃セリフ・感情描写・逆転シーンをベンチマークと比較。
5. 仏教統合: 教義の引用を日常語で1-2行に圧縮し、ストーリーに溶かす。
6. 実践セクション: 視聴者が使えるフレーズ/習慣を3つ提示し、ポジティブフレーミングで誘導。
7. クロージング: 余韻＋行動喚起＋チャンネルCTAを自然な流れで。
### 5.2 10段階ストーリーチェック（戦略.md）
1. 導入共感 → 2. 主人公紹介 → 3. 事件 → 4. 感情嵐 → 5. 教えの想起 → 6. 逆転決断 → 7. 場の変化 → 8. 後日談 → 9. 仏教解説 → 10. クロージング。
- 各段階で不足があれば、CSVの構成案やベンチマーク描写を引用して補強。

### 5.3 品質チェック指標
- 文字数: 6k-8k / 15-20分 / 音声検証済み。
- 登場人物: 名前が聴覚で判別できるか（頭子音・音節数・性別の差）。
- 語尾: 同一語尾3連続禁止、疑問・断言・余韻を織り交ぜる。
- 比喩: 1セクションにつき最低1つの視覚的イメージ。
- 感情: 「胸が熱くなる」ではなく具体的身体反応を必須化。
- 仏教引用: 一説につき最大40字。難語には即日常語訳を付ける。
