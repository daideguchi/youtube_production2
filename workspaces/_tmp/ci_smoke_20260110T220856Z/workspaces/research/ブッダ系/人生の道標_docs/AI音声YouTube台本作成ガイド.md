## 🎯 AI音声YouTube台本の5大原則

### 原則1: 読みやすさ ≠ 聴きやすさ

**根本的違い**:
```
小説（視覚メディア）:
- 主語を明示
- 複雑な文章構造OK
- 読者は戻って確認できる

AI音声台本（聴覚メディア）:
- 会話では主語省略
- シンプルな文章構造
- 聞き手は戻れない → 一度で理解必須
```

**実装方法**:
- ナレーション: 主語明示
- 会話シーン: 主語省略
- シーン転換: 主語明示必須

### 原則2: 量より質の深掘り戦略

**時間拡張の正解**:
```
❌ 悪い例: 5箇所に30行ずつ浅く追加 → 冗長・薄い
✅ 良い例: 2-3箇所に集中して深掘り → 具体的・深い
```

**深掘りの方法**:
- 抽象的説明 → 具体的シーン再現
- 「清水は傲慢だった」→ 実際に見下した場面を描写
- 対比構造の活用（過去 vs. 現在、A vs. B）
- 心理描写の段階性（認めたくない → 葛藤 → 理解 → 行動）

### 原則3: ポジティブフレーミング

**メッセージの伝え方**:
```
❌ 防御的: 「尊厳は守られ」「避けるべき」
✅ 能動的: 「格が上がり」「得られる」「成長する」
```

**理由**:
- 視聴者は「何を得られるか」に反応
- ネガティブ回避 < ポジティブ獲得
- 行動に繋がる表現

### 原則4: 完全な日常語化

**専門用語の危険性**:
```
レベル1: 明らかな専門用語（医学用語等） → 容易に発見
レベル2: 業界特有の言葉（歩留まり、節） → 発見困難
レベル3: 制作者には自然に見える言葉 → 最も危険
```

**日常語化の基準**:
- 60代の一般視聴者が理解できるか？
- AI音声で初耳でも意味が推測できるか？
- 文脈なしで単独で理解できるか？

### 原則5: 物語の流れを尊重

**見出し・構造の最小介入**:
```
❌ 8個の見出し → 物語を細切れに
✅ 2個の見出し → 大きな転換点のみ
```

**理由**:
- 物語は一気に聞かせる（読ませる）ことが重要
- 中断ポイントを作るべきではない
- 没入感 > 構造明確化

```

```
→ シーン設定・状況説明では主語必須

**会話シーン**:

❌ 田中は頷きます。

```
→ 動作→台詞のリズム重視、主語は冗長

**判断基準**:
- 直前がセリフ → 主語削除OK
- 反応描写 → 主語削除OK（相手は明確）
- シーン転換 → 主語明示必須
- 複数人登場 → 状況により判断

#### 3-2. 専門用語の日常語化

**変換例**:
```
❌ 歩留まりが悪い → ✅ 使えない部分が多い
❌ 節が多くて → ✅ 質が悪くて
❌ 慢（まん） → ✅ 心、自分を大きく見せたい気持ち
```

**チェック方法**:
1. 台本を音読（または脳内再生）
2. 初耳で意味が分からない言葉をマーク
3. 60代の親に説明する言葉に置き換え

#### 3-3. 時制・文体の統一

**基本ルール**:
- です・ます調で統一
- 過去形で統一（物語部分）
- 読点ルール遵守（一息で読める長さ）

## 🎯 AI音声最適化ルール詳細版

### ルール1: 会話シーン主語削除（最重要）

**基本原則**:
```
会話シーン = 主語省略が最適
理由: 聞き手は声と文脈で「誰」を判断する
```

**具体例**:

**悪い例（主語あり）**:
```
清水は微笑んで答えました。
「そうですね」
田中は頷きます。
「わかりました」
```

**良い例（主語なし）**:
```
微笑みました。
「そうですね」
頷きました。
「わかりました」
```

**適用パターン**:

1. **セリフ直後の動作**:
```
「ありがとうございます」
田中は深々と頭を下げます。 → 深々と頭を下げました。
```

2. **反応描写**:
```
その言葉に、清水は驚きます。 → 驚きました。
なるほど、と清水は頷きます。 → 静かに頷きました。
```

3. **動作→台詞のリズム**:
```
微笑みました。
「私も同じような経験があります」
少し俯きました。
「あの時は辛かったです」
```

**例外（主語必須）**:

1. **シーン転換**:
```
その時、田中が声をかけました。 ← 主語必須
```

2. **複数人の複雑なシーン**:
```
清水が本を差し出すと、田中が受け取り、石川が横から覗き込みます。
← 3人登場、主語必須
```

3. **予想外の動作主**:
```
すると意外なことに、石川が口を開きました。 ← 主語で驚きを強調
```

### ルール2: 専門用語の日常語化

**3段階チェック**:

**レベル1: 明らかな専門用語**
```
❌ 仏教用語: 慢（まん）、我執、煩悩
✅ 日常語: 心、こだわり、悩み
```

**レベル2: 業界用語**
```
❌ 製造業: 歩留まり、ロット、工程
✅ 日常語: 使える部分、まとまり、手順

❌ 木材業: 節、芯材、辺材
✅ 日常語: 質、硬い部分、柔らかい部分
```

**レベル3: 制作者には自然に見える言葉**
```
❌ やや難しい: 尊厳、矜持、箔
✅ よりシンプル: 価値、誇り、格
```

**判断基準**:
1. 60代の親に口頭で説明して通じるか？
2. AI音声で聞いて、文脈なしで意味が推測できるか？
3. 辞書を引かずに理解できるか？

### ルール3: ポジティブフレーミング

**変換パターン**:

**防御的 → 能動的**:
```
❌ 尊厳は守られ → ✅ 格が上がり
❌ 避けるべき → ✅ 得られる
❌ 防ぐ → ✅ 実現する
```

**受動 → 能動**:
```
❌ 見下されても → ✅ 自分を高めることで
❌ 傷つけられない → ✅ 強くなる
```

**否定 → 肯定**:
```
❌ 〜しない方がいい → ✅ 〜する方がいい
❌ 問題を起こさない → ✅ 良好な関係を築く
```

### ルール4: 深掘りの具体的手法

**抽象 → 具体の変換**:

**悪い例（抽象）**:
```
清水は経理部長で、職人を見下していました。
```

**良い例（具体）**:
```
30年前、経理部長だった清水のもとに、
石川が書類を持ってきました。

「清水さん、この資材費の件ですが」

清水は書類に目を通すこともなく、
「後で見ておくから置いといて」
と素っ気なく答えました。

内心では、
「所詮職人だ。頭を使う仕事じゃない」
そう思っていたのです。

経理こそが会社の中枢、
職人は末端。

その意識が、清水の態度ににじみ出ていました。
```

**心理描写の段階性**:
```
段階1: 現実を認めたくない
「そんなはずはない」

段階2: でも現実が突きつける
「しかし、周りの態度は明らかに変わっていた」

段階3: 徐々に理解
「もしかしたら、自分が変わったのかもしれない」

段階4: 行動変化
「清水は、石川に声をかけることにしました」
```

### ルール5: 見出しの最小介入

**配置基準**:
```
物語導入: 見出し不要
物語本編: 見出し不要（没入感優先）
物語→解説: 見出し配置 ✅
解説→実践: 見出し配置 ✅
実践まとめ: 見出し不要
```

**見出しの書き方**:
```
❌ 悪い例:
## 清水さんの物語 ← 平凡
## ブッダの教え ← 抽象的

✅ 良い例:
## 【仏教が教える「慢」を捨てる知恵】 ← 専門性とSEO
## 【今日から使える、逆転の一言】 ← 行動喚起と具体性
```

### ルール6: 語尾の多様化（修正24・最重要）

**根本問題**:
```
AI音声の致命的欠陥:
- 同じ語尾 = 同じ音程・リズム・イントネーション
- 3+連続 = 機械的・単調・聴覚的疲労
- ユーザー反応: 「すっごいイライラする」
```

**絶対禁止ルール**:
```
❌ 3+連続同一語尾 = 即座に修正必須
⚠️ 2連続でも避けることを推奨
✅ 多様化 = 聴きやすさの劇的向上
```

**5つの多様化技術**:

**技術1: 時制の混合**
```
❌ 悪い例（8連続「ました」）:
清水は答えました。
田中は遮りました。
石川は俯きました。
清水は笑いました。
田中は入ろうとしました。
石川は言いませんでした。
清水は頷いただけでした。
部屋には静寂がありました。← 8連続!!

✅ 良い例（多様化）:
清水は答える。← present る
田中は遮った。← past た
石川は俯く。← present る
清水は笑う。← present る
田中は入ろうとした。← past た
石川は何も言わず、ただ頷いただけ。← 体言止め
部屋には静寂が流れた。← past た
```

**技術2: 体言止めの活用**
```
❌ 悪い例:
清水は経理部長でした。
数字を扱うのが仕事です。
会社の頭脳だと思っています。

✅ 良い例:
清水は経理部長。← 体言止め
数字を扱う仕事。← 体言止め
会社の頭脳。← 体言止め
```

**技術3: 変化形の挿入**
```
基本形「です」の変化:
- です → なのです
- です → でしょう
- です → なのだ
- です → ですから
- です → 体言止め

基本形「ました」の変化:
- ました → た（plain past）
- ました → る（present）
- ました → 体言止め
- ました → のです
```

**技術4: 疑問形・感嘆形の挿入**
```
❌ 単調:
清水は驚きました。
その理由がわかりませんでした。
不思議に思いました。

✅ 多様化:
清水は驚く。← present
なぜか？← 疑問形
理由がわからなかった。← past た
不思議。← 体言止め
```

**技術5: 短文への分割**
```
❌ 長文連続:
清水は深く息を吐きました。そして目を閉じました。
静かに考え込みました。

✅ 短文多様化:
清水は深く息を吐いた。
目を閉じる。
静かに考え込む。
```

**実装チェックリスト**:
```
1. 全文を通して同一語尾をカウント
2. 3+連続箇所を即座にマーク
3. 5つの技術から適切な方法を選択
4. 修正後、再度カウントして確認
5. 音読して単調感がないか最終チェック

### 問題6: 語尾の単調さ（修正24・最重要）

**症状**:
```
8連続「ました」:
答えました → 遮りました → 俯きました → 笑いました →
入ろうとしました → 言いませんでした → 頷いただけでした → ありました

ユーザー反応: 「すっごいイライラする！！」
```

**なぜ問題か**:
- AI音声: 同じ語尾 = 完全に同じ音程・リズム・イントネーション
- 人間の声: 同じ語尾でも自然に変化させる（無意識的）
- AI音声の致命的欠陥: 機械的反復が脳の疲労を誘発
- 3+連続で聴覚的ストレス、視聴離脱の直接原因

**検出方法**:
```python
# 簡易チェック方法
1. 台本を縦に読む
2. 語尾だけを抽出: ました、ました、ました...
3. 3+連続 → 即座に修正マーク
4. 全文カウント → 分布を確認
```

**解決策（5つの技術）**:

1. **時制の混合**:
```
ました → る（present）
ました → た（plain past）
ました → 体言止め
```

2. **体言止めの活用**:
```
でした → 名詞で終わる
です → 名詞で終わる
```

3. **変化形の挿入**:
```
です → なのです/でしょう/なのだ
ました → のです/わけです
```

4. **疑問形・感嘆形**:
```
ました → か？/なぜか？/驚きだ！
```

5. **短文への分割**:
```
長文連続 → 短文に分けて語尾変化
```

**実装例（187.txt修正24）**:
```
Before:
Line 43: 答えました。
Line 46: 遮りました。
Line 49: 俯きました。
Line 53: 笑いました。
Line 56: 入ろうとしました。
Line 60: 言いませんでした。
Line 61: 頷いただけでした。
Line 62: ありました。← 8連続!!

After:
Line 44: 石川は静かに答える。← る
Line 47: 遮った。← た
Line 50: 石川は俯く。← る
Line 54: 鼻で笑う。← る
Line 57: 入ろうとした。← た
Line 60: 何も言わず。← 体言止め
Line 61: ただ黙って頷いただけ。← 体言止め
Line 62: その後、長い沈黙が流れた。← た

結果: 8連続 → 0連続、ユーザー満足度劇的向上
```

**絶対ルール**:
- ✅ 3+連続同一語尾 = 即座修正必須（例外なし）
- ⚠️ 2連続でも避けることを強く推奨
- ✅ 初稿段階から意識して書く（後修正より効率的）
- ✅ 音読時に「単調だな」と感じたら即修正

**効果**:
- 聴覚的疲労: 劇的減少（ユーザー実証済み）
- 視聴離脱率: 大幅低下
- 臨場感: 現在形活用で向上
- リズム感: 自然な会話調実現
