# サムネイルワークスペース改善計画

## 目的
- サムネイル管理画面をドラッグ＆ドロップ前提の直感的な UI に再設計し、作業者の手戻りを減らす。
- 既存案件の閲覧性を高め、重要情報と主要アクションに素早くアクセスできるレイアウトへ刷新する。
- エラー／処理中／完了などのフィードバックをカード単位で視認できるようにし、操作感を向上させる。

## 現状の課題
1. ドロップゾーンが既存バリアントと同列に並び、視認性・操作性が低い。
2. 案件カードが縦長で情報が散在しており、必要情報を探しづらい。
3. フィードバック（エラー・成功など）が画面上部に集約され、どの案件が対象か分かりにくい。
4. レスポンシブ時に余白と改行が崩れ、一覧性が著しく低下する。

## 目指す UX イメージ
- **カード構成**: 左ペインに概要（タイトル／担当／更新日）、右ペインにステータスや採用案、主要ボタンを配置。
- **バリアント表示**: カード下部に 2〜3 列のサムネタイルを表示。各タイルにはプレビュー・ステータス・メモ抜粋を表示。
- **アップロード導線**: カード上部にフル幅ドロップゾーン + クリック選択のボタンを配置。未登録案件は背景をトーンダウンし「未登録」バッジを表示。
- **フィードバック**: 案件ごとにスピナー／エラー／成功バナーを表示し、操作結果を即時確認可能に。
- **アクセシビリティ**: キーボード操作でドロップゾーンにフォーカスでき、Enter/Space でファイル選択を開けるよう維持。

## メモ
- 差分が大きくなるため、JSX・スタイルの変更は段階的に行い、タスク完了ごとに本ファイルの状態を更新する。
- バリアントタイルは後で個別コンポーネントに切り出す余地があるが、現段階では同ファイル内で管理する。
- 動作確認の際は `npm run build` に加え、`npm start` で実際のドラッグ＆ドロップ操作を確認する予定。
- UI 上に保存先パス（`workspaces/thumbnails/assets/{CH}/{video}/`）を常時表示済み。今後はワンクリックコピーや戦略メモ欄などの拡張も検討する。
