## 1) 最重要ルール（音声品質の下限）

### 1.1 ポーズ挿入の唯一の記号
- **ポーズ挿入として許可される記号は `---` のみ**。
- `---` は **1行単独**で置く（前後は改行）。意味の切れ目（話題転換/場面転換/呼吸の区切り）にだけ使う。
- 原則: **セクションが切り替わる境界**（設計図/構成上の区切り）で `---` を置く（TTS側の0.5秒ポーズに対応）。
- 音声側の解釈（strict）:
  - 通常のつなぎは **0.1秒**（デフォルト）
  - `---` は **0.5秒**の無音ポーズ
  - 空行/改行はポーズ指示として扱わない（整形のために入れてもよい）
- `***` や `___`、`///`、`===` 等の区切り記号は使用しない（音声の不自然な途切れや事故の原因）。

### 1.2 記号・括弧の過剰使用を禁止（TTSが不自然に切れやすい）
- `「」` / `『』` と `（）` は **多用しない**（必要な場面だけ）。原則として **直接話法より間接話法** を優先する。
- 引用はできるだけ `「」` / `『』` を使わず、地の文で言い換える（例: “ある人はこう言いました” の形）。
- 目安（TTS事故防止）: 1本あたり `「」` / `『』` は **合計20個（10組）以内**、`（）` は **合計10個（5組）以内**。超える場合は意味を保ったまま言い換える。
- 運用上の扱い: この「目安」は **script_validation のハード上限**として扱う（超過は不合格）。
  - 上限は `configs/sources.yaml` の `script_globals.a_text_quote_marks_max` / `script_globals.a_text_paren_marks_max` が正本（必要ならチャンネル別override可）。
  - 超過した場合は **自動クリーナ（非LLM）で括弧だけ外して基準内に落とす**（意味は維持・全文リライト禁止）。
  - その結果、字数が下限を割った場合は **Extend/Expand（追記のみ）で回復**する（リトライ回数で押し切らない）。
- 会話の連打でテンポを作らない。台詞が必要なら **短く**、地の文に溶かし込む。
- ルビ/読み仮名/注釈目的の括弧は原則禁止（読み上げが破綻しやすい）。

### 1.3 視聴者ストレスを生む文章を禁止
- 淡々とした繰り返し構文、同じ言い回しの連発、蛇足の比喩、回りくどい前置き、過剰な煽りを避ける。
- 断定・説教・脅し・人格否定はしない。必要なら事実と提案に落とす。
- “言い換えで水増し” をしない。厚みは **具体の追加**で作る（状況・行動・心の動き・結果）。
- **同一段落の丸ごと重複は禁止**（コピー/生成ループ由来の反復）。  
  目安（機械チェック）: 120文字以上の段落が同一（空白差を無視して一致）なら不合格。

### 1.4 捏造・虚構の禁止（信頼を壊さない）
- 根拠のない統計・割合（`％` / `パーセント`）・研究結果・大学名/研究所名・論文/実験の作り話を本文に書かない。
- 事実確認ができない固有名詞や数値は使わず、必要なら一般化して述べる（例: “研究では〜” の断定や数字は避ける）。
- 科学/医学の橋渡しは、**安全な一般論を短く1段落**に留める（呼吸と緊張、書くことと客観視など）。数字・機関名・論文名は出さない。

### 1.5 「字数合格だけ」を禁止（LLM品質ゲート必須）
- 文字数や禁則がOKでも、内容が薄い/冗長/流れが不自然なら **不合格**とする。
- フロー/自然さの合否は **機械的ルールではなく推論モデルの判定（LLM Judge）**で担保する。
- 運用の正本: `ssot/ops/OPS_A_TEXT_LLM_QUALITY_GATE.md`（Judge→Fixer の2段階で最終品質を固定）。

### 1.6 終わり方（未完を禁止）
- Aテキストは **最後まで完結**していること（途中で切れて終わるのは不合格）。
- 目安（機械チェック）:
  - 最後の有効文字が `。` / `！` / `？` / `!` / `?` / `」` / `』` などで閉じている。
  - 助詞や語尾の途中（例: `〜になり` / `〜で` / `〜が`）で止まらない。
- これは「決まり文句を入れろ」ではなく、**ぶつ切り事故（生成途中で止まる）を確実に検知して止める**ためのルール。

### 1.7 文字の健全性（外字/制御文字を禁止）
- `�`（U+FFFD, Unicode replacement character）が混入している場合は不合格（コピー事故/文字化けの典型）。
- 目に見えない制御/整形文字（例: ゼロ幅スペース等）や、私用領域（PUA）等の特殊文字が混入している場合は不合格。
- 明らかな誤字・外字が検出された場合は不合格（TTSの読み崩れ/視聴者ストレスの原因）。  
  既知の混入パターンは `script_validation` の決定論クリーナで **置換修復**する（全文リライトは禁止）。

## 2) Aテキストの禁止事項（混入事故を根絶）

台本本文（Aテキスト）に入れない:
- URL（`http(s)://` / `www.`）
- 出典メタ、脚注、参照リンク、引用番号（例: `[13]` / `([xxx][13])`）
- 箇条書き/番号/記号リスト（`-` / `・` / `1.` / `A.` 等）
- アウトライン由来のメタ（例: `約600字`、`A. 見出し - 約600字` などの「字数メモ」）
- Markdown装飾（例: `**太字**`）
- 設定/CSVデータ/構成案/プロット等の **制作メモの貼り付け**（例: `設定` 見出し＋ `主人公: ...` の羅列）
- エピソード見出し行（例: `CH23-001: ...`）
- タイムスタンプ、チャプター表、制作裏（AI/収録/台本などのメタ言及）
- 章立てメタ（例: `第3章` / `第3章を始めましょう` のような“章見出し”）

※出典は本文ではなく `content/analysis/research/references.json` 等の research 側へ集約する。

---

## 3) 読みやすさ（耳で理解できる最小設計）

- 1段落は1話題。2〜4文を目安に詰め込みすぎない。
- 1文は長くしすぎない（息継ぎできる長さ）。主語が迷子にならないように整える。
- 専門語は最小限。出すなら直後にやさしい言い換えを添える（定義の長話はしない）。
- 比喩は “一度で伝わる短さ” に限定し、連打しない。

## 4) チャンネル別ルールとの優先順位（衝突時の扱い）

1. 本書（全チャンネル共通のAテキスト品質）  
2. チャンネル固有の `script_prompt.txt` / Persona / 企画CSV  
3. 作業メモ・一時指示

補足:
- チャンネル `script_prompt.txt` に構成/形式/記号の指示が混ざっていても、Aテキスト向けLLM呼び出しでは衝突しやすい要素を落とした `a_text_channel_prompt`（派生）を用いる。骨格は `ssot/ops/OPS_SCRIPT_PATTERNS.yaml` が正本。

例外が必要な場合:
- チャンネル固有の理由と例外条件を **SSOTに明記**してから適用する（無断で例外を増やさない）。


【対象コンテンツ】
一見平和な絵手紙サークルに、なぜか気疲れを感じ始めたのはいつからだったでしょうか。

毎週木曜の午後、公民館の和室に集まるのが楽しみでした。窓から差し込む柔らかな光。畳の香り。広げた画用紙と色鉛筆の音。最初はただ、好きなものを描いて、隣の人に見せて笑い合うだけの時間でした。桜の季節にはそれぞれが思い思いの桜を描き、夏には朝顔や金魚鉢。自分のペースで、ゆっくりと手を動かす。それだけで、一週間のちょっとした憂さが晴れていくような気がしたものです。

そのサークルには、代表の和子さんという方がいました。初めは、進行役として優しく声をかけてくれる、そんな存在でした。場所の予約をしてくれたり、お茶の準備をそつなく整えたり。みんなから自然と信頼され、何かあれば和子さんに相談するのが当たり前の空気が、いつの間にか生まれていました。

次第に、その存在感が少しずつ、形を変えていったように思います。例えば、作品の題材です。ある時、和子さんが「今月は秋の果物をテーマにしませんか」と提案しました。それ自体は悪いことではありません。けれど、それ以降、毎月のように和子さんからテーマの提案があり、いつの間にかそれがその回の「お題」のようになっていったのです。別のものを描きたいと言う人もいませんでした。皆、にこにことそのお題を受け入れ、柿や栗、ぶどうを描き始めます。

展示の順番もそうです。公民館の廊下に作品を飾る時、最初はくじ引きで決めていたのが、次第に和子さんが並び順を考えるようになりました。この人の作品はここに、あの人のあれはここと。彼女のセンスで決められることが、当然の流れになっていきました。誰も異論を唱えません。和子さんが決めてくれるのだから、きっとそれが一番良い並び方なのだろうと、みんなが思っているようでした。

そして、ほんの少しずつ、メンバー同士の会話に、ある気遣いが混じり始めます。和子さんが今、どんな作品を気に入っているか。次のテーマを何にしたいと考えているか。彼女が機嫌よくしている時と、少し口数が少ない時。そんなことが、お茶を飲みながらの何気ない雑談の中で、さりげなく話題に上るようになりました。直接的に「気を遣おう」と言うわけではない。ただ、空気としてそこに漂い始めたのです。

誰かが和子さんの好みとは少し違う色使いをした時、周りが一瞬、その作品を見て沈黙する瞬間がありました。あるいは、和子さんが「この部分、もう少し練習したらもっと良くなるわよ」と、特定の人の作品にだけ微笑みながら言う。その場は和やかなままだけれど、後でその人が少し俯き加減になる。そんな小さなことが、一度や二度ではなく、繰り返し起こるようになっていきました。

悪意があるわけではない。和子さん自身、サークルを良くしようと真心から動いているのでしょう。メンバーも、波風立てずに楽しい時間を過ごしたいだけ。それなのに、なぜか木曜日が近づくにつれて、心のどこかに小さな石を抱えているような、そんな感覚が生まれ始めていました。公民館へ向かう道すがら、今日はどんな空気だろうかと、考えてしまう自分がいました。

変わらぬ日常のはずなのに、その中で、ほんの少しずつ、息を潜めるような緊張が芽生え始めていたのです。

---

なぜ、ささいなことが気になり始めるのでしょう。最初はほんの小さなことでした。絵手紙サークルが終わった後、ふと胸に引っかかる何か。帰り道、なぜか少し重い足取り。それは、大きな喧嘩や決定的な悪意ではなく、毎週のように繰り返されるちょっとした気遣いの積み重ねから生まれていました。

例えば、お茶当番の順番のこと。これまでくじ引きで決めていたのが、ある日から和子さんが指名する形に変わりました。特に理由は告げられず、ただ今日はあなたでお願いね、と軽く言われるだけ。断る理由も見当たらないので、みなさんはいえいえ、と引き受けます。でも、次週もまた指名され、その次も。順番が公平でなくなったことが、少しずつ意識の片隅に留まります。

作品についてのコメントもそうです。サークルでは各自が描いた絵手紙を並べて、互いに鑑賞する時間があります。和子さんは代表として、それぞれに言葉を添えます。ほとんどのメンバーには、素敵ですね、この色がいいですね、と褒めるような感想。ところが、ある人の作品の前でだけ、少し長く立ち止まり、もっと背景を練習した方がいいわね、とだけ言って去ることがありました。その人は、にっこり笑ってはいはい、と答えます。しかし、その場の空気が一瞬、ひんやりとするのを感じ取る人は少なくありません。

和子さんからの依頼は、それだけではありません。公民館の窓拭きを誰か手伝ってくれないか、次の作品展の案内状を数枚余分に書いておいてほしい、そんな小さなお願いが、休憩時間の合間にさりげなく投げかけられます。大きな負担ではないから、とみなさんは快く引き受ける。引き受けることが、自然な流れになっていきました。

断りにくい空気。それが、次第にサークルの中に漂い始めます。和子さんが不機嫌そうに黙っている日は、なぜか誰も大声で笑わない。お菓子の差し入れをする人が、和子さんの好みを選ぶようになる。はっきりと指示されるわけではない。ただ、これまでと違うことをすると、和子さんがそっとため息をつく。そのため息が、まるで小さな鐘のように響いて、次からはやめておこうという気持ちにさせるのです。

冷たい視線、というほど露骨なものではありません。でも、和子さんが誰かをじっと見つめる時、その目は少し評価しているような、測っているような印象を与えます。例えば、お茶当番を代わってほしいとお願いしたメンバーがいたとします。和子さんは了承しますが、その後しばらく、その人には話しかけません。別のメンバーが話しかけようとすると、和子さんがそっとその腕に触れて、今忙しいから後でにしましょう、と取り成す。そんなことが一度や二度あれば、みなさんは自然に学びます。和子さんの意向に沿わないことは、控えた方が良いのだと。

気遣いが習慣化する様子は、少しずつ染み込んでいきます。集まりの前に、今日は和子さんの機嫌はどうかな、と無意識に考えるようになる。自分の作品を並べる時、これは和子さんにどう見られるだろうか、と一瞬ためらう。発言する前に、周りの反応を伺う目線が、自分の中に根付いていく。最初は自分だけかもしれない、と思っていたことが、実は他のメンバーも同じようにしていると気づく瞬間があります。お互いに、そっと目配せするだけ。言葉には出さない共通の了解が、そこに生まれています。

なぜ、こんなにささいなことが気になってしまうのでしょう。一つひとつは、本当に小さなことです。お茶の順番が変わったくらいで、何が問題なのか。作品に注釈をつけられても、それは指導の一環かもしれない。頼みごとをされるのは、信頼されている証拠だと言えなくもない。しかし、それらが毎週のように繰り返され、少しずつパターン化していく時、それは単なる偶然や気まぐれではなく、その場の決まり事のように感じられてきます。その決まり事は、明文化されていない。口に出されていないからこそ、余計に気を遣い、神経を使う。

小さな棘は、皮膚に刺さるとすぐには痛みを感じません。でも、動くたびにじわりと突き刺さり、いつの間にか違和感が疼き始める。サークルの時間が終わった後、家に帰ってほっとした時に、ふとその日の小さな出来事が頭をよぎる。あの時、ああ言えば良かったかな。もっとうまく対応できたのではないか。そんな後悔に似た感覚が、少しずつ心のエネルギーを削っていきます。

楽しみだったはずの絵手紙の時間が、知らず知らずのうちに気苦労の多い場に変わり始めています。それでも、みなさんは顔を合わせれば笑顔で挨拶を交わす。作品を見せ合えば、やはり素敵ですね、と褒め合う。表面は、以前と何も変わらない平和なサークルの風景。ただ、その裏側で、それぞれが小さな棘を数えながら時間を過ごしている。そんな毎週が積み重なっていきました。

こうした小さな積み重ねは、目立たないけれど確実に心に影を落とします。それは、やがて集団の中にいながら、なぜか一人ぼっちのような感覚を募らせていく土台となるのでした。

サークルには人がいる。笑い声も、おしゃべりも、いつもと同じようにそこにある。なのに、なぜか一人ぼっちのような気がするときがあります。みんなの輪の中にいて、それでも、どこか遠くに取り残されたような。この感覚は、いったいどこから来るのでしょう。

サークルの前日、水曜日の夜になると、少し憂鬱な気分がやって来ました。明日は木曜日、公民館の日です。特に用事があるわけではない。天気も悪くはない。それなのに、心のどこかが重たい。明日のことを考えると、そっとため息が出る。何かが怖いというわけではないのです。ただ、少し、疲れるのだなと感じる。明日の準備をしながら、その気持ちをかき消そうとします。絵具を揃え、筆を洗い、半紙を広げる。いつもの作業なのに、手の動きがどこか遅い。

そして当日。公民館へ向かう道すがら、足取りは自然と重くなります。あの和室の扉を開ける前に、一呼吸置くことが癖になっていました。中からは、もう誰かの声が聞こえています。和子さんの明るい笑い声です。さあ、入らなくては。そう自分に言い聞かせて、顔を上げ、口元を緩めます。いつの間にか、この一連の動作が決まり事のようになっていました。

和室の中は、いつも通りの光景です。長い机に、メンバーがそれぞれの場所に着いています。和子さんは中央の席で、皆の作品に次々と目を通していました。あそこに座ると、自然と目が集まる。彼女が笑えば、周りもそれに合わせて笑う。彼女が黙ると、誰かが慌てて話題を探す。空気の流れが、そこを中心に回っているのが、はっきりと感じられました。
