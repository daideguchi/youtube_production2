---
■ 0) まず読む（SSOT）
- `ssot/ops/OPS_CONFIRMED_PIPELINE_FLOW.md`（確定フロー/SoT）
- `ssot/ops/OPS_ENTRYPOINTS_INDEX.md`（叩くべき入口）
- `ssot/ops/OPS_AUDIO_TTS.md`（TTS運用）
- `ssot/plans/PLAN_OPS_ARTIFACT_LIFECYCLE.md`（生成物の保持/削除）

---
■ 1) 事前条件（絶対）
- `script_validation` が `completed` の動画だけ処理する（未完は必ずフェーズ1へ差し戻し）。
  - 確認: `python -m script_pipeline.cli status --channel CHxx --video NNN`
- **1本ずつ** 処理する（並列禁止）。失敗したら次へ進まない。

---
■ 2) 実行（確定入口）
1. 音声・SRT生成（正本の入口）
   ```bash
   python -m script_pipeline.cli audio --channel CHxx --video NNN
   # 途中から再開したい場合:
   # python -m script_pipeline.cli audio --channel CHxx --video NNN --resume
   ```

2. 出力の確認（SoT）
   - 正本出力先: `workspaces/audio/final/{CH}/{NNN}/`
   - 必須:
     - `{CH}-{NNN}.wav`
     - `{CH}-{NNN}.srt`
     - `log.json`（生成ログ）

3. status.json の補正（stage反映）
   ```bash
   python -m script_pipeline.cli reconcile --channel CHxx --video NNN
   python -m script_pipeline.cli status --channel CHxx --video NNN
   ```
   - `audio_synthesis` が `completed` になっていることを確認する。

4. 追加チェック（任意だが推奨）
   ```bash
   python -m script_pipeline.cli validate --channel CHxx --video NNN
   ```

---
■ 3) エラー時の対応（迷わない）
- 途中で止まったら「勝手に直さない」。
- まずログ/症状を保存し、必要なら pending artifact を埋めて再実行する（THINK/AGENT mode）。
  - 申し送り: `python3 scripts/agent_org.py memo --to '*' --subject "audio failed CHxx-NNN" --body "<要約 + コマンド + エラー抜粋>"`
- `script_validation` 未完で止まった場合は必ずフェーズ1へ戻す（本文修正は禁止）。

---
■ 4) 完了後の掃除（残骸を残さない）
目的: `audio_prep/` の chunks や final と重複する中間バイナリを消し、混乱と容量を防ぐ。正本は `workspaces/audio/final/`。

- まず dry-run:
  ```bash
  python -m scripts.cleanup_workspace --channel CHxx --video NNN --dry-run
  ```
- 問題なければ実行:
  ```bash
  python -m scripts.cleanup_workspace --channel CHxx --video NNN --run
  ```

---
■ 5) 作業記録（必須）
- 実行したコマンド・所要時間・結果（成功/失敗/再実行理由）を memo に残す（追記方式）。
  - 例: `python3 scripts/agent_org.py memo --to '*' --subject "audio done CHxx-NNN" --body "..."`。
