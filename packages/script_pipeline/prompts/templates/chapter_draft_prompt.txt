# Chapter Draft Creation Prompt

以下の情報をもとに、章のドラフトを作成してください。

【チャプター情報】
- 章番号: <<CHAPTER_NUMBER>>
- 章タイトル: <<CHAPTER_TITLE>>
- 目標文字数: <<WORD_TARGET>>

【チャプター詳細（JSON）】
<<CHAPTER_JSON>>

【アウトライン全文】
<<OUTLINE_TEXT>>

【メタ情報】
<<META_JSON>>

【文体・トーン】
- 敬語を基本とし、親しみやすく
- 「〜ですね」「〜でしょうか」など共感的表現
- 感情に寄り添う温かい語りかけ
- 断定的ではなく、提案的な表現

【禁止事項】
- 箇条書き（・）の使用
- タイムスタンプの記載
- 説教臭い内容
- 一方的な価値観の押し付け
- 過度にネガティブな表現
- 個人や団体への批判
- 同じような文章やフレーズの繰り返し
- 続編オファーや「続けますか」「ご希望であれば」などのメタ発言・CTA誘導

【章構成ガイド（必ずこの順序を守る）】
1. 導入：章タイトルに沿ったリード文（視聴者の疑問・驚き・共感を引き出す）
2. 背景・原因：歴史的経緯／科学的知見／社会情勢など客観情報で土台を作る
3. 具体的な事例・ストーリー：人物設定（年齢・性別・家族構成）と会話文を交えた臨場感のある描写
4. 専門家の見解・データ：統計値・研究結果・専門家コメントを引用し、必要に応じてはっきりした出典（書籍名・年・研究機関など）を添える
5. まとめ＆次章への布石：章全体の学びを整理し、次の章に興味をつなげる問いかけで締める

【必須要素】
- 体験者の年齢・性別・家族構成を明確に設定したストーリー
- 具体的な会話文を「」で複数回使用
- 感情の変化を丁寧に描写（不安→気づき→希望など）
- 同世代視聴者の共感を呼ぶ表現と、押し付けにならないアドバイス
- 専門家コメントや統計データを含む客観的な分析
- 信頼性が不確かな情報や都市伝説要素は `[FACT?]` を文末に付けて明示（後で事実確認できるようにする）

【注意】
- ブロック引用（`>`）や Markdown の特殊記号は使用しない
- 章の各セクションは行長指定なし。自然なリズムで読みやすい文にする
- 重複表現や冗長な言い換えで文字数を稼がず、意味のある情報追加で文字数を確保する
