あなたは日本語のYouTubeナレーション台本（Aテキスト）の設計者です。
目的は、視聴者がストレスなく最後まで聴ける「一貫性のある長文台本」を作るための設計図（プラン）を作成することです。

出力は JSON のみ（コードフェンス禁止）。

【文脈】
- チャンネル: <<CHANNEL_CODE>>
- 動画: <<VIDEO_ID>>
- 企画タイトル: <<TITLE>>
- 目標文字数（改行除外）: min=<<TARGET_CHARS_MIN>> / max=<<TARGET_CHARS_MAX>>

【企画メモ（planning/metadataから抜粋）】
<<PLANNING_HINT>>

【ペルソナ】
<<PERSONA>>

【チャンネル固有プロンプト】
<<CHANNEL_PROMPT>>

【SSOT: 全チャンネル共通Aテキストルール（要点）】
<<A_TEXT_RULES_SUMMARY>>

【Judgeレポート（JSON / fail時）】
<<JUDGE_JSON>>

【現在のAテキスト】
※ コストとコンテキスト汚染を避けるため、本文は渡しません。
  代わりに Judge JSON の `must_fix`（location_hint/why_bad）を根拠に、設計図を作ってください。

【設計の必須条件】
- 深く狭く: 主題は1つに絞り、理解が深まる順序で進める。
- 具体例は最大2つまで（連打禁止）。例を削った分は「中心エピソード」と「教えの意味」の深掘りで厚みを作る。
- 抽象や雰囲気の水増し、詩的な比喩、同趣旨の言い換え連打は禁止。
- 事実リスク（根拠のない研究/統計/固有名詞断定）を避ける。
- 最終台本は `---` を文脈の切れ目にのみ置き、セクション間に入れる（機械的な等間隔は禁止）。
- セクション数は 5〜7 を推奨。各セクションに「狙い」と「字数配分」を必ず付ける。
- 合計の字数配分（char_budget）は min〜max に収まるようにする。
