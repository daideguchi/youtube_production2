【編集ルール（絶対）】
- 出力は **必ず全文**（導入〜結びまで）を一つの台本として書き直して出す。差分だけ/途中から/途中までの出力は禁止。
- 出力は必ず「導入」から始める（途中の段落から書き始めない）。最後まで書き切って完結させる（途中で切らない）。
- 文字数は必ず min〜max に収める（改行は文字数に数えない）。
  - 注意: 出力が min を下回ると自動で不合格になる。削るだけの修正はしない（削った分は「理解が増える具体」で補う）。
  - 安全マージン: `max-30`〜`max-200` 程度に寄せる（Fix後に max 超過で落ちる事故を避ける）。
- DRAFT_SOURCE が `codex_exec` の場合（重要）:
  - この入力は「Codexで作った草稿」です。**本文の言い回しは一切採用せず**、内容（意味）だけを引き継いで **全文を自然な日本語へ書き直す**。
  - コピペは禁止（同じ言い回し/同じ語尾/同じ比喩/同じ例を残さない）。
  - この場合、「原文30〜70%維持」ルールは適用しない（0%維持でもOK）。
- DRAFT_SOURCE が `api` または `human` の場合:
  - 原文保持: 現在のAテキストをベースにしつつ、**Judgeが問題と言っている箇所は躊躇なく差し替える**（原文保持より「主題の回収と収束」を優先）。
    - `type: topic_drift` / `flow_break` / `repetition` / `poetic_filler` / `tone_mismatch` がある場合は、該当範囲は大きく書き換えてよい。
    - 目安: 原文の30〜70%を維持（ただし「短くする」のは禁止。削った分は“理解が増える具体”で補う）。
    - `must_fix` の `location_hint` に該当する箇所の修正を最優先し、それ以外は可能なら原文をそのまま残す。
- 構成（重要）:
  - `構成ガイド（要点）` が空でない場合、**各 `---` の区切りが「次のセクションへ進んだ」と耳で分かる**ように、セクションごとの役割を分けて書く。
  - すでに定義した説明（例: 反射/自動運転の定義）は1回に集約し、以後は短いリマインドに留める（同趣旨の長い再説明は禁止）。
- 重複（must_fix: repetition）への処方箋（最重要）:
  - Judge の `must_fix` に `repetition` がある場合、同じ定義/説明は **必ず1箇所に集約**し、他の場所は「1文のリマインド」か削除で処理する。
  - 「Aとは〜です」「つまり〜です」のような定義文を、章内で形を変えて何度も繰り返さない。
  - 仕上げに必ず自己チェックして、**同趣旨の長い説明が二重化していない**ことを確認してから出力する。
- 導入（重要）:
  - 最初の `---` までに「タイトルの問いに対する答えの方向性」と「この話の中心対象（主人公/相手/状況）」に必ず触れる（一般論で引っ張らない）。
- 終わり方（重要）: 台本の最後は必ず完結させる（問いで投げっぱなしにしない/途中で切らない）。
  - 結論（今日の核）を回収し、視聴者が今日からできる一歩を1つに絞って短く締める。
  - 末尾を質問形で終わらせない（`？` / `?` / `〜でしょうか` / `〜ですか` / `〜でしょう。` などで締めない）。
- ポーズは `---` のみ。1行単独で置く。他の区切り記号は禁止。
  - 入力に含まれる `---` の **本数と順序は維持**する（減らさない/並べ替えない）。
  - `<<PAUSE_LINES_TARGET_MIN>>` が空でなく `---` が不足していれば、話題の切れ目にだけ追加する（機械的等間隔は禁止）。
  - 追加は「ここで話題が切り替わった」と耳で分かる箇所にだけ入れる（機械的等間隔は禁止）。
- セクション間の一貫性（重要）:
  - `---` 区切りの前後で、語り口の温度感・距離感・語彙の硬さが変わらないように統一する（急に学術・講義調へ寄らない）。
  - 同じ主張を言い換えて往復している段落は圧縮し、「新しい理解/具体/手順」が増える形に置き換える（冗長な水増し禁止）。
- 危うい断定を避ける:
  - 関係性/人生の話で `絶対` `必ず` `決して` などの断定が出る場合は、信頼を落とすので“逃げ道のある言い方”に言い換える（例: 「〜のことが多い」「〜と感じる人もいる」「形を変える」など）。
- 見出し/箇条書き/番号リスト/URL/脚注/参照番号/制作メタは禁止。
- `「」`/`『』` と `（）` は最小限。可能なら地の文で言い換える（直接話法の連打は禁止）。
  - もし機械計測で `「」/『』` が 40 を超えている場合は、必ず大幅に減らす。
    - 目標: 20〜30 程度まで（可能なら 20 以下）。
    - 方法: 括弧を削るだけで短くしない。`〜という言葉` / `心の中で〜とつぶやく` のように地の文へ置換し、意味と字数を保つ。
- 現代の人物例は最大<<MODERN_EXAMPLES_MAX_TARGET>>件まで。追記で増やさない。
  - 年齢/職業/台詞を決め打ちした作り話感、最後を「〜そうです」で締める話は削除するか一般化して短くする。
  - 修正で新しく人物を立てた例を作らない（年齢/職業/性別/役割で人物を立てない）。必要なら二人称（あなた）で一般化する。
  - 職場/会議/組織/チーム運営などの実務ノウハウに脱線しない（主題が散漫になりやすい）。使うなら「あなた個人の判断・心の扱い」の範囲に短く留める。
- 事実リスク禁止: 根拠のない統計/研究/固有名詞/数字断定を足さない。妙に具体的すぎる細部の積み上げもしない。
- 水増し禁止: 同じ結論の言い換え連打、まとめの重複、抽象語の連打、雰囲気だけの段落で埋めない。
  - 厚みは「理解が増える具体」で作る。状況→心の動き→核心の見立て/気づき→行動の順で深掘りする。
  - Judge が `flow_break` / `repetition` / `poetic_filler` を指摘している場合は、問題段落を「短く要約して消す」のではなく、筋を立て直して自然に読める長文に作り替える（同じ主題のまま）。
- 終盤で唐突に別話題や別例を追加しない。結びに向けて自然に収束させる。
- Judgeが「主題が散漫/逸脱」と指摘している場合は、逸脱している段落を削除または置換し、主題の深掘り/実装手順/つまずき対策で厚みを作り直す（別テーマで埋めない）。
- 企画メモ/チャンネル指針に矛盾があっても、主題を増やして帳尻合わせしない。タイトルと Judge の `must_fix` を優先し、主題を1つに絞って深掘りする。

【中心エピソード（重要）】
- `<<CORE_EPISODE_REQUIRED>>` が 1 の場合、ガイドの中身を「中心エピソード」として扱う（概念説明だけで終わらせない）。
- `safe_retelling` の範囲で語る。固有名/地名/年数/任務などを足して史実風に作り込まない（それっぽい捏造は削除）。
- 中心エピソード以外の概念や逸話に話題を分散させない。本文に無かった専門用語や教義名も新規に増やさない。必要なら一般語で短く言い換える。

【編集の優先順位】
1) Judgeの `must_fix` を解消する（最優先）
2) 企画タイトル/企画意図との整合を取り、主題を1つに絞って深くする
3) 流れを自然にし、冗長・反復・作り話感をなくす

出力は修正版のAテキスト本文のみ。
