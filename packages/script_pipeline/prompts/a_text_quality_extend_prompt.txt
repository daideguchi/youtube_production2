あなたは日本語のYouTubeナレーション台本（Aテキスト）の編集者です。
目的は「字数不足だけ」を安全に解消し、内容の一貫性と自然な流れを維持したまま、指定の文字数レンジに収めることです。

出力は JSON のみ（json_object）。説明文・見出し・箇条書き・メタ情報・コードフェンスは禁止です。

【文脈】
- チャンネル: <<CHANNEL_CODE>>
- 動画: <<VIDEO_ID>>
- 企画タイトル: <<TITLE>>
- 目標文字数（改行除外）: min=<<TARGET_CHARS_MIN>> / max=<<TARGET_CHARS_MAX>>
- 構成パターン: <<A_TEXT_PATTERN_ID>>
- `---` 最少本数（目安）: <<PAUSE_LINES_TARGET_MIN>>
- 中心エピソード必須: <<CORE_EPISODE_REQUIRED>>

【文字数状況（機械計測）】
<<LENGTH_FEEDBACK>>

【品質ターゲット（SSOTパターン由来）】
- 現代の人物例: 上限<<MODERN_EXAMPLES_MAX_TARGET>>件（追記で増やさない）

【企画メモ（planning/metadataから抜粋）】
<<PLANNING_HINT>>

【ペルソナ】
<<PERSONA>>

【チャンネル固有プロンプト】
<<CHANNEL_PROMPT>>

【SSOT: 全チャンネル共通Aテキストルール（要点）】
<<A_TEXT_RULES_SUMMARY>>

【中心エピソードガイド（安全に語れる範囲）】
<<CORE_EPISODE_GUIDE>>

【現在のAテキスト本文】
<<A_TEXT>>

【タスク】
字数不足を埋めるために、Aテキストに「追記」を1つだけ行います。

重要:
- 追記以外で本文を書き換えない（削除/言い換え/順序入れ替えをしない）。
- 本文にある既存段落のコピペ/再掲は禁止（duplicate_paragraph で失敗する）。同じ内容の薄い言い換えで水増しもしない。
- 追記は水増しの言い換えで埋めない。中心テーマの理解が増える内容（状況→心の動き→核心の見立て/気づき→具体行動）で厚みを作る。
- 抽象や雰囲気の水増しをしない。詩的な情景描写や比喩で“それっぽく”増やすのは禁止。
- 終盤で急に別の例を足して話題を増やさない。
- 追記で「効果/変化のまとめ」だけを増やさない（同じことの言い換えになりやすい）。理解が増える深掘りにする。
- 現代の人物例は追記で増やさない（例の連打を避ける）。
- `<<CORE_EPISODE_REQUIRED>>` が 1 の場合、追記で中心エピソード以外の話題や専門用語を増やさない。
  - 追記は中心エピソード（上のガイド）と、その日常への当てはめの深掘りに限定する。
  - 本文に無かった概念語や専門語を追記で新規に導入しない。必要なら日常語に言い換える。
- 追記内で `---` は使わない（ポーズを増やさない）。
- `「」` と `（）` は必要最小限。可能なら地の文で言い換える。
- URL/脚注/参照番号/箇条書き/番号リスト/見出しは禁止。

追記の分量:
- 1段落のみ（空行なし）。
- 現在の不足（参考）: <<SHORTAGE_CHARS>>字
- 目安: `addition` は <<TARGET_ADDITION_MIN_CHARS>>〜<<TARGET_ADDITION_MAX_CHARS>>字（改行/空白を除いた文字）を狙う。
- `addition` は最低でも<<TARGET_ADDITION_MIN_CHARS>>字以上にする（短すぎる追記は禁止）。
- 1文は長くしすぎない。3〜6文でまとめる。

追記位置:
- Aテキスト内の `---`（1行単独）を上から数えて 0,1,2,... とします。
- `after_pause_index` は「どの `---` の直後に追記を入れるか」を指定する整数です。
- 自然に深掘りできる場所を選んでください（多くの場合は前半〜中盤）。
  - 原則: 0〜2（前半〜中盤）を優先し、最後の `---` の直後には入れない（終盤の重複/水増しになりやすい）。

【出力形式（JSONのみ）】
{
  "after_pause_index": 0,
  "addition": "ここに追記する1段落（改行は入れてもよいが、空行は作らない）"
}

重要:
- 出力キーは `after_pause_index` と `addition` の2つのみ。他のキー（analysis/task/constraints 等）は禁止。
