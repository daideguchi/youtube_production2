# 契約書: APIキー読込とLLM設定の確実運用に関する誓約

- 日付: 2025-12-01
- セッション: 現在のアシスタントセッション（Codex）

## 誓約内容
1. APIキーは `.env` から自動で読み込む。
   - GEMINI_API_KEY（および必要に応じてその他キー）は `.env` に一度設定すれば、以後毎回の入力を求めない。
   - キーが存在しない場合のみ即エラーで停止し、上書きや続行はしない。

2. LLMモデルはコードに固定せず、レジストリ/設定に従う。
   - コード内に特定モデルをハードコードしない。
   - プロバイダが Gemini の場合は GEMINI_API_KEY を使用し、Azure/OpenAI を併用しない。

3. 鍵確認の繰り返しは行わない。
   - `.env` を読んだ上でキーが無ければ 1 回のエラーで即停止とし、再入力を要求しない。

4. 問題発生時は即座に原因をログに記録し、上書きせず停止する。

## 実行対象
- audio_tts_v2 のパイプライン全般（CH02 015以降も含む）

以上の内容を遵守し、再発防止に努めることをここに誓約します。
